Color Palette




Design Decisions

Color:

  • I chose the colors dark purple and beige because they strongly contrast each other. Additionally, purple is a gender neutral color, applicable to both baby boys and girls.


Repetition:

  • All text colors are white, and the card layout are consistent with one another.


Proximity:

  • Individual controls of the strollers are grouped by their own respective cards.


Alignment:

  • Contents that can be appropriately aligned to the left are left aligned, to emphasize the line of alignment which enhances readability.


Overlay for disconnect action

  • Helps minimize disconnect error by requesting for confirmation.


Simplified home page with all controls located in one place:

  • Improves the accessibility of the stroller controls.


Labels on control cards:

  • Signify function of controls.


Use of world knowledge for lock:

  • Lock color is red, unlock color is green. Helps the user easily identify the state of the lock.



Interaction Analysis

Music Player Control


Purpose:

  • The play and pause button, the skip button, and the back button allow the user to control the music playing.

Affordance:

  • The buttons afford pressing by the user to control the specific functions acted on the music playing.

Labels:

  • The buttons are not labeled, but instead in the form of shapes. This is ideal because these controls are universally known and consistent in other music player platforms. Since it is best to avoid labels when possible, these buttons are ideal.

Body language:

  • Because the music control buttons are easily discoverable as they are on the home page, not complex because there is only three buttons, and easily learnable since these buttons are universally known, the body language is not difficult at all to use.

Pin Input Box


Purpose:

  • Allows the user to input the pin code on their stroller to manually connect their device to their stroller.

Affordance:

  • The input text box affords number values to be inputted into the text box.

Labels:

  • The input text box has “XXX-XXX-XXX” labelled which specifies the way the pin code should be formatted when it is inputted in.

Body language:

  • The sizing of the text box is just the right size to indicate the length of characters to be inputted. The text box is not too large that it makes the user assume to input a lot of information, and not too short that it creates the assumption that the input length is short.

Light Slider


Purpose:

  • Control brightness of stroller headlights.

Affordance:

  • Affords light and visibility around stroller when it gets dark.

Labels:

  • Flashlight icon lets the user know this slider controls a light source on the stroller.

Body language:

  • Sliding will turn on lights which is immediate feedback.

Lock Button


Purpose:

  • Lock and unlock the storage compartment in the stroller.

Affordance:

  • Affords control of the safe mechanism on the stroller. Also affords safe storage on the stroller for the user to store their items in a theft-proof manner.

Labels:

  • The lock incon signifies a lock and unlocking feature of the stroller.

Body language:

  • The default position will be locked to ensure that thiefs do not breach the system if the user forgets to lock their safe.

Disconnect Button


Purpose:

  • Bring the user to the scan QR code screen when they disconnect.

Affordance:

  • Button affords stroller to be disconnected.

Labels:

  • Disconnect label tells the user that clicking on it will disconnect the stroller.

Body language:

  • When clicked, a pop-up will appear for the user to confirm the disconnection to ensure commitment and prevent error.





Testing


To test my prototype, I utilized usability testing by developing 5 tasks for the user to try and complete using the prototype. Afterwards, I will prompt the user a quick survey in order to gain more insight about their user experience. I critiqued my design based on Shneiderman's 8 Golden Rule.



Shneiderman's 8 Golden Rule:

  • Strive for consistency

  • Enable shortcuts

  • Offer informative feedback

  • Indicate closure

  • Prevent and recover from the error

  • Permit reversal of actions

  • Give user control

  • Reduce short-term memory load



Tasks

  1. Find the instructions on how to connect a stroller

  2. Connect to the stroller using the manual pin input method

  3. Turn on the headlights

  4. Unlock the storage compartment

  5. Disconnect the stroller
